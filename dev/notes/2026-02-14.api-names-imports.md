API: Names and Imports
==============================================================================

A sketch of how to clean the API surface up.

-   `splatlog.setup.setup` gets moved into `splatlog/__init__.py` to simply
    become `splatlog.setup`.

-   `splatlog.rich_handler.RichHandler` in `splatlog/rich_handler.py` moves to
    `splatlog.rich.Handler` located in `splatlog/rich/handler.py`.
    
    This works because `splatlog.rich` is an _opaque_ module with an entry in
    `autodoc2_module_all_regexes` in `docs/conf.py`,

-   `splatlog/named_handlers.py` stays where it is, but functions have
    `named_handler(s)` removed from their names, and the whole `named_handlers`
    module is re-exported from `splatlog/__init__.py`.
    
    It's used like

    ```python
    import splatlog

    splatlog.named_handlers.get("custom")
    splatlog.named_handlers.set("custom", custom_factory)
    ```

    or like

    ```python
    from splatlog import named_handlers

    named_handlers.get("custom")
    named_handlers.set("custom", custom_factory)
    ```

    Basically, usage is always off the module, which provides namespacing,
    getting rid of the "double namespacing" annoyance of
    `splatlog.named_handlers.get_named_handler`.

    The `splatlog.named_handlers.named_handler` decorator would need to be
    renamed, maybe `splatlog.named_handlers.factory`. `del_named_handler` might
    want to become `delete`, because `del` is a built-in. `check_name`, 
    `to_console_handler`, and `to_export_handler` stay the same, but those are
    mostly internal functions.

    I like this, because I like structural namespacing much more than
    pre/suf/midfix namespacing, but I'm a bit hesitant because I feel like this
    pattern is not seen much in Python, and that may be for a good reason..?

-   Remove `splatlog.names.root_name` — it's not even used — and move
    `is_in_hierarchy` into `splatlog/levels/filter.py` where it's actually used,
    to get rid of the `splatlog/names.py` file all-together.

-   `splatlog/splat_logger.py` becomes `splatlog/logger.py` gets the same
    treatment as `named_handlers`, resulting in calls like
    `splatlog.logger.get`. Move the `getLogger` alias into
    `splatlog/__init__.py`. Keep all the class names the same.

-   I'd like to rename `splatlog/typings.py` to `splatlog/typing.py`, and
    re-export the whole `splatlog.typing` from the package root instead of each
    symbol. I know that there can be some trouble with stdlib name shadowing,
    but we have `splatlog.json` and that has seemed to be just fine.

-   `splatlog/locking.py` is fine as-is, but it doesn't need it's symbols or the
    entire module re-exported from the package root, I doubt dependents will use
    it much if ever.

-   `splatlog/report.py` gets renamed `splatlog/reporting.py`, and just the
    `report` function gets moved up to `splatlog/__init__.py`, with the rest not
    re-exported.
